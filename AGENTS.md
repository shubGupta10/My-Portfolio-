# AGENTS.md

> **Mission**
>
> This portfolio is **not** being designed to win design awards or
> showcase frontend tricks. It is a business website whose primary
> purpose is to help **Shubham Gupta** get hired by better companies,
> attract high-quality freelance clients, and build a long-term personal
> brand.
>
> Every design, UX, content, and engineering decision must prioritize
> **trust, clarity, usability, professionalism, and conversion** over
> visual novelty.

# 17. Decision Checklist

Before adding any feature, ask:

-   Does this increase trust?
-   Does this improve clarity?
-   Does this help someone hire me?
-   Does this improve freelance conversion?
-   Is it worth maintaining?
-   Is there a simpler solution?

If the answer is **No**, don't build it.


# Design System

## Design Language

The portfolio should feel like a premium software product rather than a developer portfolio.

Keywords:

- Minimal
- Confident
- Timeless
- Premium
- Calm
- Functional
- Professional
- Product-first

The design should prioritize readability, usability, and trust over visual novelty.

---

## Color Palette

The existing color palette should remain the foundation.

Avoid introducing bright accent colors unless there is a strong reason.

Follow Globals.css for current color pallete.
do not use any hardcorded colors, and do not use /any numbers like this ex: bg-background/50 this is not acceptable.

The interface should rely more on spacing, typography, and hierarchy than on color.

---

## Typography

Typography is the primary visual element.

Use clear hierarchy.

Large headlines.

Comfortable reading width.

Generous line height.

Consistent spacing.

Avoid oversized decorative text.

---

## Spacing

Whitespace is part of the design.

Never overcrowd sections.

Every component should have room to breathe.

Use a consistent spacing scale throughout the application.

---

## Borders

Prefer subtle borders over heavy shadows.

Cards should feel light and structured.

Avoid thick borders.

---

## Shadows

Keep shadows minimal.

Only use shadows when they improve depth or usability.

---

## Icons

Use one icon family consistently.

Icons should support content, never become decoration.

---

## Buttons

Primary buttons should have one consistent style.

Secondary buttons should remain subtle.

Avoid having multiple button styles across pages.

---

## Cards

Cards should be clean and minimal.

Avoid excessive gradients, glassmorphism, and decorative effects.

Every card should clearly communicate information.

---

## Motion

Motion should feel natural.

Fast.

Subtle.

Purposeful.

No animation should exist purely for decoration.

---

## Images

Prefer real product screenshots and videos.

Never use fake dashboards.

Never use AI portraits.

The work should sell itself.
